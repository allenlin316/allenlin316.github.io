搜尋與排序

  • 搜尋 (Search)
    • Linear Search
      • Time Complexity : O(n)
    • Binary Search
      • Time Complexity : O(logn)、O(1) (best case)
  • 排序 (Sort)
    • Insertion Sort
      • Time Complexity : O(n) (best case)、O(n^2) (worst case)、O(n^2) (average case)
      • stable sort
    • Selection Sort
      • Time Complexity : O(n^2) (best、average、worst case)
      • unstable sort
    • Bubble Sort
      • Time Complexity : O(n) (best case)、O(n^2) (worst case)、O(n^2) (average case)
      • stable sort
    • Quick Sort
      • Time Complexity : O(nlogn) (best case)、O(n^2) (worst case)、O(nlogn) (average case)
      • unstable sort
    • Merge Sort
      • Time Complexity : O(nlogn) (best、average、worst case)
      • stable sort
    • Heap Sort
      • Time Complexity : O(nlogn) (best、worst、average case)
      • unstable sort

Pipeline 公式推導

  • 定義 ：將電腦指令處理過程拆分為多個步驟，作出類比，則電腦指令就是管線傳送帶上的產品，各個硬體處理單元就是管線旁的工人，每個工人都負責自己的事情，把管線SOP化
  • Concept
    • 假如有四個工人分別做fetch(FE)(10ns)、decode(DE)(5ns)、execute(EX)(10ns)、writeToMemory(WM)(10ns)
    • 第一段時間裡 : FE的工人開始工作(第一條指令)
    • 第二段時間裡 : FE的人做完了接著做第二條指令的，同時DE也開始它的第一條指令(5ns)
    • 第一段時間需耗費5ns、第二段時間需耗費10ns
  • 基礎計概46題

Memory Management

  • Contiguous (連續性)
  • Non-Contiguous (不連續性)

I/O Processing

  • Programmed I/O (Polling)
    • CPU一直監督I/O是否完成，直到I/O設備準備好
  • Interrupt-driven I/O
    • CPU先發出I/O命令，然後去做別的事，直到I/O準備好了就Interrupt CPU
  • Direct memory access (DMA)
    • 從disk→memory間會有一個DMA controller 暫存部分data
    • CPU 發出input/output的訊息、記憶體的初始位址、有多少的位元組需要被傳送訊息告訴DMA
    • 當 I/O device準備好了的話，告訴CPU需要匯流排的控制權，CPU轉移匯流排給DMA
    • DMA與memory 結束檔案傳輸之後，還給CPU

RAID (Redundant Array of Independent Disk)

  • 降低CPU和Disk的access time，確保資料安全和可靠性
  • RAID 0

    效能導向，速度最快，data striping

  • RAID 1

    浪費一半的空間存相同的data(mirror)，可靠性高(fault tolerant)

  • RAID 0+1

    兼具fault tolerance、speed up

  • RAID 3

    有disk專門存parity bit，bit 為切割單位(效能差)

  • RAID 4

    以block為切割單位，有disk存parity bit

  • RAID 5
    • parity bit分開存放，改善RAID3、4效能瓶頸
    • 儲存成本、資料安全、儲存效能兼顧
  • RAID 6
    • 需經過 parity 容錯機制，需要一顆硬碟負責容錯機制

Data Compression

  • Data compression 有兩種方式 : lossless methods v.s. lossy methods
  • Lossless Methods (非破壞性資料壓縮)
    • Run-length
      • 只是用頻率去代換
    • Huffman
      • 轉換成二元樹結構
    • Lempel Ziv
      • 創造字典再代換
  • Lossy Method
    • JPEG
    1. 把照片分成很多的 blocks (用以減少計算量)
    1. 透過 DCT (discrete cosine transform) 方法，每個blocks中的每個pixels的關係被存起來，同時找到幾乎沒改變的資料(redundancies)
    1. change the pic into a linear (vector) set of numbers
    1. 把 redundancies (lack of changes) 會透過其中一種無損壓縮刪除

范紐曼架構

  1. 為內儲程式的概念(stored programming)，即欲執行的程式跟資料，會先載入到memory中再循序執行
  1. 把電腦分成四個子系統，分別為memory、arithmetic logic unit、control unit、input/output

范紐曼瓶頸 (Von Neumann bottleneck)

  • 范紐曼架構將CPU與記憶體分開 。當CPU要從記憶體抓指令過來解碼跟執行，之後再把結果寫回記憶體，會一直耗費時間在處理存取的問題，尤其是當需要頻繁存取記憶體的時候，會明顯拖慢效能－也就是「范紐曼瓶頸」
  • Overall : 當CPU與memeory效能不斷增加時，傳輸速度會成為整體效能上的瓶頸

Register

  • Data Register : 放input data & result of operation
  • Instruction Register : CPU 從 memory 拿到的指令放在這裡
  • Program Counter : 指向下一個欲執行指令的位址
  • Address Register : 用來存記憶體位址的暫存器，可以用來存取記憶體 (e.g. CPU 必須先將要存取位址存入位址暫存器，才能到主記憶體中存取資料)
  • 通用目的暫存器 : 可以用來存資料或位址(同時是資料暫存器和位址暫存器)

Optical Storage Devices

  • CD-ROM : CD-ROM 比 CD 更耐用、可以檢查錯誤 (Hamming code) 單倍速的標準讀取速度為 150KB/S
  • CD-R : 常用為備份，不能複寫，can write once to CD-R disks, but they can be read many times (WORM)，讀取速度跟CD-ROM一樣
  • CD-RW : 可以復寫，空的 blank CD-RW 比 blank CD-R貴，讀取速度跟CD-ROM一樣
  • DVD (digital versatile disk) : CD-ROM (650MB) 不夠大，DVD可以儲存最少4.7GB
  • Blu-ray Disk : DVD之後的下一代光碟格式之一，單層就有25GB

Cache v.s. Memory

  • 區塊(block/line) : cache與memory間傳遞資料的單位
  • 主記憶體是靠 位址 存取，而快取則是靠 內容 存取，所以又稱為內容定址記憶體(content addressable memory)
  • 內容定址快取中的 內容 即 主記憶體位址的子集 ，稱為 欄位(field)
  • 字組欄(Word Field)(offset) : 特定區塊中的字組
  • 欄位是記憶體位址的一部份，有多對一的對映關係
  • 記憶體中的多個區塊會對應到快取中的同一區塊，快取中的 tag欄位 會區分出是哪一個區塊
  • 快取的對映機制有 3 種
    • Direct-Mapped cache
      • 主記憶體的區塊只能放入快取中的某一特定區塊

      https://www.youtube.com/watch?v=pSarQQTJbDA

    • Fully associative-mapped cache
      • 主記憶體區塊可以放置於快取中的任一位置
      • 找尋資料時必須搜尋快取中的全部資料

      https://www.youtube.com/watch?v=Pg6bqkekAXY

    • n-way set associative-mapped cache
      • 用位址將區塊對映到快取位置，並非對映到單一區塊，而是一些區塊的集合

      https://www.youtube.com/watch?v=isbdr73Ymug

Endianness (位元組順序)

  • 資料在記憶體中的放置順序
  • 有兩種位元組順序 : 大端序、小端序
  • Big-Endian
    • 最高位的位元組放在最低的記憶體位址

  • LIttle-Endian
    • 最高位的位元組放在最高的記憶體位址上

電腦開機程序

  • A small section of memory is made of ROM and holds a small program called the bootstrap program. 當開機時，電腦會開始跑bootstrap 這個程式，藉由這段程式可以找到系統的核心程式並將他載入記憶體，然後開始執行核心程式
