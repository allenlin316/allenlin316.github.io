ch10- Software Engineering

  • Software lifecycle (3 processes)
    • Development、Use、Modification
    • Use & Modification 一直會重複執行，直到軟體過時 (使用者需求的改變、軟體沒有效率等)
  • Development process models
    • Waterfall model
      • 整個 development process 固定一個方向 (即前一個步驟做完，才能進下一個步驟)
      • 優點
        • 執行design時能清楚知道要做什麼 (因為已經完成了 analysis phase)
        • 能夠對整個系統測試，因為已經做完了前面的所有 phases
      • 缺點
        • 在 testing 時，有問題的時候必須全部找一遍
    • Increment model (stepwise refinement)
      • 每次做完大致的架構，就 testing ，沒問題就在加新功能
  • Analysis phase
    • Procedure-oriented
      • Data flow diagrams : 呈現系統中的 data flow
      • Entity-relationship diagrams
      • State diagrams : 當狀態隨著事件發生而改變，會用這個圖 (如: 電梯)
    • Object-oriented
      • Use case diagram
      • Class diagram
  • Design phase
    • Structure chart : 把東西分成一個個模組 (modules)
      • Modularity
  • Implementation phase
    • Software quality factors
      • Operability : basic operation of a system
      • Maintainability : system can be kept up to date and running correctly
      • Transferability : ability to move data or system from one platform to another
  • Testing phase
    • Glass-box testing (white-box testing)
      • Basis path testing : each statement in the software is executed at least once
      • Control structure (比 basis path testing 還全面)
        • Condition Testing
        • Data flow testing
        • Loop testing
    • Black-box testing
      • Exhaustive testing
      • Random testing
      • Boundary-value testing
  • Documentation
    Documentation is an ongoing process
    • User documentation (使用說明書)
    • System documentation (s.t. software can be maintained & modified by people other than original developers)
    • Technical documentation (how to download、how to maintain)
