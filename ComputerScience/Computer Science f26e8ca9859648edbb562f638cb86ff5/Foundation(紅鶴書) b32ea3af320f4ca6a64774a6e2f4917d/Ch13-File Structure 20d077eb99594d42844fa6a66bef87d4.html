Ch13-File Structure

檔案結構的分類

Sequential File

  • 更新sequential file
  • 作業系統只知道整個檔案存在哪裡，不知道每一個 record 的位址
  • New master file : contains the most current data
  • Old master file : 一些舊檔案應該要被上傳更新的，即使更新完舊檔案會被留下來做成 reference
  • Transaction file : 存放一些要對檔案做處理的資料 (如 : Add、Delete、Change)，要做到增加、刪除、更改的交易 (transaction) 需要用到 key
  • key : 由一或多個欄位(field)組成，可以在檔案中區分出唯一資料
  • Processing file updates
    • if master file key > transaction file key → transaction 對 new master 做處理
    • if master file key == transaction file key → transaction 對 master file 做處理
    • if master file key < transaction file key → old master file write to new master file

Indexed Files

  • 由 data file 、 index 組成
  • index (key+address)
    1. 整個indexed file 丟進memory裡
    1. 透過algo.找到key
    1. 透過key得到address，就可以得到對應的資料
  • Inverted Files : 不只一個index，即不只一個key

Hashed Files

  • 不用index的支援
    • 透過數學函式，得到key對應的address
  • 不同的hashing 方法
    • Direct hashing
      • 沒有 synonyms 、 collisions
    • Modulo division hashing
      • 公式 : key % file size+1
    • Digit extraction hashing
      • selected digits are extracted from the key and used as the address
      • e.g. 有一個6位數的key，要hash成三位數的address，取第一、三、四位做Digit extraction hashing
        • Ans : 125870 → 158、122801 → 128
  • synonyms、collision
    • synonyms：有兩筆key經過雜湊值運算後得到相同的位址
    • collision : 如果有一個key值經過雜湊後得到的位址，是 已經 有資料的，則稱為碰撞
  • home address、prime area
    • home address ：由hashing algo. 得到的位址
    • prime area：The part of the file that contains all the home addresses，檔案中包含所有原始位址的部份
  • collision resolution
    • open addressing
      • 會提高未來碰撞的機會
    • Linked list
    • Bucket hashing
      • 可能會有很多用不到的地方
  • 良好雜湊函數性質
    1. 計算簡單，不要過度複雜
    1. 碰撞要少
    1. 避免資料群聚，要均勻才好

Text v.s. Binary files

  • Text file
    • 檔案裡面只有字元，不能有其他的檔案格式(檔案內容也為一序列的二元資料)
  • Binary file
    • 可以儲存圖檔、音樂檔、自己定義結構的檔案
    • 一檔案內容為一序列的二元資料或程式的可執行碼
