網路硬體

  • Switch
    • 在 data-link layer 使用 MAC 位址轉發資料，引入路由功能，可以在 network layer 轉發資料
    • 只會將資料轉發到需要接收的裝置與 HUB 不同
  • Bridge (橋接器) (網橋)
    • 將網路的多個網段在資料鏈路層（OSI模型第2層）連接起來（即橋接）
    • 橋接器有別於路由器。路由器允許多個網路之間的獨立通信，但又保持區隔，而橋接器則是將兩個獨立的網路連接起來，就如同單一網路
  • Gateway (閘道器)
    • 連接兩個異質網路
    • 為應用層的裝置

Overview

Network : 設備間可以互相連接及溝通

Internet : 兩個以上的 network 連接在一起，可以互相溝通

Protocol : 指在網路上一種可以彼此溝通的模式

Router : 一個網路連接到下一個網路

modem : 改變 data 的形式

switch : 把設備連接在一起

  • WAN (Wide Area Network)
    • point-to-point WAN
    • Switched WAN
  • Addressing & packet names
    • 要建立雙方連接需要 : source address、destination address
    • Application Layer : 是透過names (e.g. google.com)
    • Transport Layer : 利用port number，只要運行的程式向系統提出訪問網路的申請，那麼系統就可以從這些Port號中分配一個供該程式使用
    • Network Layer : 網路位置 ( the connection of a device to the Internet)
    • Data-link layer : 由 MAC address

Application Layer

  • logical conection 來連接 (假想兩方連接只要一個send、一個receive)
  • Client-server paradigm
    • HTTP(port 80)、FTP、SMTP、SSH、WWW
    • WWW v.s. HTTP
      • hypertext (超連結) : 這個概念在 Internet 之前就出現了
      • web page : 為一個 file with name & address
      • Simple web page : 沒有任何連結到其他 web pages
      • Composite web page : 有一個以上的連結到其搭 web pages
  • Web client (browser) (瀏覽器)
    1. controller : 利用 client 的 program 去取得網站的東西
    1. client protocols : HTTP、FTP、SMTP
    1. interpreters : controller 利用任何的直譯器( JavaScript、HTML、Java) 呈現網站
  • URL (Uniform resource locator)
    1. Protocol : 首先先告訴瀏覽器我們需要哪一種的 client-server application
    1. Host : IP address of the server
    1. Port : 16-bit integer，只要運行的程式向系統提出訪問網路的申請，那麼系統就可以從這些Port號中分配一個供該程式使用
    1. Path : location and the name of the file，格式依作業系統不同
  • FTP (File transfer protocol)
    • 當FTP開始，Control connection 會開始連線
    • Data connection 資料傳送結束就會關掉
    • FTP 關掉，control connection 才會斷掉
  • SMTP
    1. User Agent (UA) : Allen透過UA將郵件寄給MTA
    1. Mail Transfer Agent (MTA) : Allen's MTA 寄給 Bob 的 MTA
    1. Message Access Agent (MAA) : MAA 把 MTA 的訊息 pull 下來
  • TELNET
    • remote logging
    • 所有的 password、logging name 都沒被加密的傳回去
  • SSH (Secure Shell)
    • SSL/TLS v.s. SSH

      有 CA 的參與也是 SSL/TLS 與 SSH 最大的差異。另外一個差異是 SSL/TLS 是被使用在資料傳輸，而 SSH 是用來執行指令（通常就是用來登入遠端電腦）

    • SSH-1 被 SSH-2 取代
    • SSH 目的是要取代 TELNET
  • DNS
    • 由 tree 建構而成
    1. Generic domains : 對應到申請者性質是什麼 (com、edu、gov)
    1. Country domains : 國家的縮寫
    1. Inverse domain : 原本可以透過IP 位址，找到相對應的主機，但現在太多網路，所以已經不太行找到了

Transport Layer

  • Process-to-process communication
    • A process 為應用層跑程式時會用到傳輸層的服務
  • Port numbers
    • 整數範圍 : 0~65535(16 bits)
    • local host 與 remote host 透過 IP 聯繫，透過 port numbers 建立 processes
    • client 有 ephemeral port (臨時埠) (號碼大於1023)
    • server 有 well-known port (統一的埠號)
  • UDP (User Datagram Protocol)
    • connectionless、unreliable
    • 8 bytes fixed-size header
    • e.g. : DNS、DHCP、VoIP
  • TCP (Transmission Control Protocol)
    • connection-oriented、reliable protocol
    • 它利用 sequence numbers to define the order of the segment
    • Segment : TCP groups a number of bytes together into a packet 稱之
    • TCP 在 每一個 segment 前面加上 header(20~60 bytes)，再給 network layer 做傳輸
    • 這些 segments 都被封裝在 IP-datagram 裡
    • e.g. : HTTP、HTTPS、FTP、SMTP、Telnet
  • GeeksforGeeks links
    • https://www.geeksforgeeks.org/differences-between-tcp-and-udp/

Network Layer

  • host-to-host delivery
  • 傳輸層的 payload 可能會被封裝到 network-layer 不同的封包裡
  • Packet delivery
    1. Unreliable : 封包可能會不見、壞掉
    1. connectionless : network layer 對每個封包處理都是獨立的，即由同一個 transport layer 來的 payload 可能到達 destination 的順序不一樣
  • IPv4 (Internet Protocol Version 4)
    • Address notation (base 2、base 256、base 16)

  • IPv6 (Internet Protocol Version 6)

Data-Link Layer

  • Node-to-node
  • Nodes & links
    • Node : two end hosts、routers 為nodes
    • Link : networks 在兩個 nodes 間稱為 links
  • Ethernet frame (Wired LANS)
    • 一堆 bits 被整合成一個 frame 傳送 (dest. add.、source add.、data type、data、CRC)
  • Wireless Ethernet & Bluetooth
    • Bluetooth : 它是 ad hoc network (無線臨時網路)，是一種分散式無線網路系統，它不需要依賴一個既存的網路架構，它每一個節點，都有能力轉送網路封包給其他節點
  • DSL (Digital subscriber line)
    • over the existing telephone 可以支援更快速的連接
    • ADSL (asymmetric) : downstream direction (Internet → resident) 速度大於 upstream (resident→Internet)，使用 ADSL 的人可以同時使用 data & voice channel
  • WiMAX (Worldwide Interoperability for Microwave Access)
    • 高速無線資料網路的標準
    • 主要用於都會網路
    • wireless version of DSL or Cable connection
    • 可以想成大型 WIFI

Physical Layer

  • Node-to-node 但它是交換電的訊號
  • 把 data-link layer 的 bits 轉換成 electromagnetic signals
  • Guided media
    • Twisted-pair cable (雙絞線) : DSL (電話線)
    • Coaxial cable (同軸電纜) : 之前TV是用這個，不過最近被光纖取代
    • Fiber-optic cable (光纖) : 骨幹網可以看到它的存在，因為頻寬 cost-effective
