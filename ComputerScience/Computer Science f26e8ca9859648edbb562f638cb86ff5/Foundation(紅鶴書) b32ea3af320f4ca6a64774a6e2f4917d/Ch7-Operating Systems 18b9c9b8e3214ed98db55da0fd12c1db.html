Ch7-Operating Systems

  • Operating system (作業系統) : 提供使用者與硬體之間的溝通介面，幫助其他程式的運行及取得硬體、軟體的資源
  • 作業系統的發展
    1. Batch system
    1. Time-sharing system
    1. Personal systems
    1. Parallel systems
    1. Distributed systems
    1. Real-time systems
  • 作業系統架構
    • Components of O.S.
    • User Interface : 在UNIX的作業系統裡是叫shell，Windows 中叫 window
    • Memory Manager
      • monoprogramming : 大多數的記憶體容量配置給單一程式使用，僅有一小部分給作業系統
      • multiprogramming : 同時間在記憶體中有多個程式，CPU 透過不斷在程式間切換，使得他們被 concurrent 地執行
        • 多元程式類別圖
          • Nonswapping : 程式在執行時一直留在 memory 裡
          • Swapping : 程式在執行時，會在 memory & disk 之間做交換

        • Partitioning (Nonswapping)
          • memory 被切割成 variable-length sections
          • 每個 section 存一個 program
          • 每個程式完整的在 memory 中且占有連續的位址，提高 CPU 的效率
          • 會有外部碎裂、透過 memory manager 事前管控 partitions
        • Paging (Nonswapping)
          • a page is loaded into a frame in memory
          • 非連續性配置
          • 一個程式需要6個 frames ，但 memory 中只有4個 frames 可以使用，這樣這個程式就 不能 被 load into memory
          • 會有內部碎裂
        • Demand Paging (Swapping)
          • page 可以被 load into memory one-by-one, executed, and replaced by another page

        • Demand Segmentation (Swapping)
          • program is divided into segments that match programmer's view
          • 把 program 切成 Main program & Subprogram，再 load into memory
          • 因為 memory 中的 segment 是一樣大小的，所以 program 傳入時會有一些empty 的地方
        • Virtual Memory
          • 一個程式3MB有10個，有一個 memory 10MB
          • 透過 virtual memory ，每一次都10MB在 memory，20MB 在 disk
          • 可以說是 actual memory size : 10MB，virtual memory size : 30MB
    • Process Manager
      • Process、Job、Program 的關係圖
      • 使得 process 與 job 之間的狀態改變，透過 job scheduler or process scheduler
        • Job scheduler :
        1. hold state to ready state (從 job → process)
        1. running state to terminated state (process → job)

        • Process scheduler (都是在process中) :
        1. running state to waiting state ( 需要 I/O)
        1. waiting state to ready state (I/O 結束)
        1. ready state to running state (CPU 準備好要跑process)
        1. running state to ready state (process 可以用的時間已經結束)
      • Job control block & Process control block
        • 為一小段 memory 中存關於job或process的資訊
      • Deadlock
        • Mutual Exclusion : 一資源於同一時間只允許一個 process 使用，絕不會有大於等於兩個 process 共用
        • Resource Holding : process 擁有部分資源並且又在等待其他資源
        • No Preemption : process 不能奪取其他 process 所持有的資源
        • Circular Waiting : 所有的 process 跟資源組成一個 loop
      • Differences between deadlock & starvation
    • Device Manager ( I/O manager)
      • 負責處理 process 的 I/O
    • File Manager
      • 管理 files 的一些東西 (權限存取、增刪改的更動...)
  • UNIX & Linux
    • Linux 是類 UNIX 架構
    • Utilities : 像 ls(list) 等指令
    • Kernel : 存放作業系統的基本東西，像 device manager 等
